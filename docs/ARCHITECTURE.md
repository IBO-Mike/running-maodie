# 跑酷猫咪项目架构

## 依赖方向

`MainWindow -> Pages -> GameController -> Systems / Entities`

- `app/`：程序窗口和页面切换。
- `pages/`：主页、游戏页等完整页面。
- `game/`：开始、暂停、逐帧更新和结束等游戏流程。
- `entities/`：玩家、障碍物、收集物的数据与行为。
- `systems/`：碰撞、生成和计分等独立规则。
- `managers/`：图片、音乐和音效等共享资源。
- `ui/`：分数、生命值等游戏内界面组件。
- `config/`：窗口尺寸、重力、速度等可调参数。
- `resources/`：图片、声音和字体，统一登记到 `resources.qrc`。

## 推荐实现顺序

1. 游戏场景、地面和持续移动的背景。
2. 玩家显示、跳跃和下落。
3. 障碍物生成、移动和销毁。
4. 碰撞与游戏结束。
5. 分数、HUD 和重新开始。
6. 图片动画、声音、设置和最高分。

## 模块边界

页面不直接计算碰撞或分数；实体不负责切换页面；系统不创建窗口。
这样每次增加功能时只需要修改相关模块。
