# 跑酷猫咪项目代码逐句解释

这份文档用于帮助你从 Qt/C++ 初学者视角理解当前项目代码。  
阅读顺序建议：

1. 先看 `CMakeLists.txt`，理解项目如何被构建。
2. 再看 `src/main.cpp`，理解程序从哪里启动。
3. 再看 `MainWindow`、`HomePage`、`GamePage`，理解页面如何切换。
4. 最后看 `GameController`、`GameCanvas`、`Player`、`PlayerAnimator`，理解游戏运行逻辑。

本文采用“代码写法 / 语法含义 / 项目作用”的方式解释，不完整复制每个文件的全部源码，避免文档和源码重复过多；如果你改了源码，可以按本文结构继续补充。

## 一、C++ 和 Qt 中常见语法先说明

### 1. `#include`

```cpp
#include <QApplication>
#include "app/mainwindow.h"
```

- `#include` 是 C++ 预处理指令，用来引入其他头文件。
- 尖括号 `<...>` 通常表示引入库文件，例如 Qt 的 `QApplication`。
- 双引号 `"..."` 通常表示引入项目自己的文件，例如 `app/mainwindow.h`。

项目作用：  
当前文件想使用某个类，就必须先包含声明这个类的头文件。

### 2. 头文件保护

```cpp
#ifndef MAINWINDOW_H
#define MAINWINDOW_H
...
#endif
```

- `#ifndef` 表示“如果还没有定义过”。
- `#define` 表示定义一个宏。
- `#endif` 表示条件编译结束。

项目作用：  
防止同一个 `.h` 文件被重复包含，导致类被重复定义。

### 3. 类声明

```cpp
class MainWindow : public QMainWindow
{
    Q_OBJECT
public:
    explicit MainWindow(QWidget *parent = nullptr);
private:
    QStackedWidget *pageStack;
};
```

- `class MainWindow` 定义一个类。
- `: public QMainWindow` 表示 `MainWindow` 继承 Qt 的 `QMainWindow`。
- `Q_OBJECT` 是 Qt 信号槽系统需要的宏。
- `public` 表示外部可以访问。
- `private` 表示只能类内部访问。
- `QStackedWidget *pageStack;` 表示声明一个指针成员变量。

项目作用：  
`MainWindow` 是游戏主窗口，它继承 Qt 主窗口能力，并管理首页和游戏页。

### 4. 构造函数初始化列表

```cpp
MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
    , pageStack(new QStackedWidget(this))
{
}
```

- `MainWindow::MainWindow` 是 `MainWindow` 的构造函数实现。
- `::` 是作用域解析符，表示这个函数属于 `MainWindow` 类。
- 冒号 `:` 后面是初始化列表。
- `QMainWindow(parent)` 调用父类构造函数。
- `new QStackedWidget(this)` 在堆上创建对象，`this` 表示当前窗口作为父对象。

项目作用：  
创建窗口时，同时创建页面容器。

### 5. Qt 信号槽 `connect`

```cpp
connect(startButton, &QPushButton::clicked,
        this, &HomePage::beginGameTransition);
```

- `connect` 把一个对象的信号连接到另一个对象的函数。
- `startButton` 是发出信号的对象。
- `&QPushButton::clicked` 是按钮被点击时发出的信号。
- `this` 是接收对象。
- `&HomePage::beginGameTransition` 是点击后执行的槽函数。

项目作用：  
点击“开始游戏”按钮后，触发首页到游戏界面的过渡动画。

### 6. Lambda 表达式

```cpp
connect(homePage, &HomePage::startGameRequested,
        this, [this](double backgroundOffset) {
    pageStack->setCurrentWidget(gamePage);
    gamePage->startGame(backgroundOffset);
});
```

- `[this]` 表示 Lambda 内部可以访问当前对象的成员变量。
- `(double backgroundOffset)` 表示这个匿名函数接收一个 `double` 参数。
- `{ ... }` 中是信号触发后执行的代码。

项目作用：  
首页动画结束后，切换到游戏页，并把首页背景偏移量传给游戏页，实现背景无缝衔接。

### 7. `emit`

```cpp
emit startGameRequested(backgroundOffset);
```

- `emit` 是 Qt 的语义标记，表示发出一个信号。
- 信号本身不直接做事，谁连接了它，谁就会响应。

项目作用：  
首页通知主窗口：“可以进入游戏了”。

### 8. `override`

```cpp
void paintEvent(QPaintEvent *event) override;
```

- `override` 表示这个函数重写了父类的虚函数。
- 如果函数签名写错，编译器会报错。

项目作用：  
Qt 会在窗口需要重绘时自动调用 `paintEvent`，我们在里面绘制背景、猫咪、金币和障碍物。

### 9. `constexpr`

```cpp
inline constexpr int WindowWidth = 1100;
```

- `constexpr` 表示编译期常量。
- `inline` 用在头文件中的全局常量，避免重复定义。

项目作用：  
统一管理窗口大小、重力、跳跃速度、金币尺寸等游戏参数。

## 二、`CMakeLists.txt`：项目构建说明

位置：`CMakeLists.txt`

### 关键代码解释

```cmake
cmake_minimum_required(VERSION 3.19)
```

要求 CMake 最低版本是 3.19。

```cmake
project(running-maodie LANGUAGES CXX)
```

定义项目名为 `running-maodie`，使用 C++ 语言。

```cmake
find_package(Qt6 6.5 REQUIRED COMPONENTS Core Widgets)
```

查找 Qt6，最低版本 6.5，必须包含 `Core` 和 `Widgets` 模块。

- `Core` 提供基础对象、信号槽、定时器等。
- `Widgets` 提供窗口、按钮、标签等 UI 控件。

```cmake
qt_standard_project_setup()
set(CMAKE_AUTORCC ON)
```

- `qt_standard_project_setup()` 使用 Qt 推荐的默认构建设置。
- `CMAKE_AUTORCC ON` 让 CMake 自动处理 Qt 的 `.qrc` 资源文件。

```cmake
qt_add_executable(running-maodie
    WIN32 MACOSX_BUNDLE
    src/main.cpp
    ...
)
```

创建可执行程序 `running-maodie`。  
括号中的 `.cpp`、`.h`、`.qrc` 都会参与项目构建。

```cmake
file(GLOB PLAYER_FRAME_FILES CONFIGURE_DEPENDS
    "${CMAKE_CURRENT_SOURCE_DIR}/resources/images/player/*.png"
)
```

扫描 `resources/images/player` 下的所有 `.png` 图片。  
`CONFIGURE_DEPENDS` 表示图片列表变化后，CMake 会重新配置。

```cmake
qt_add_resources(running-maodie "player_frames"
    PREFIX "/"
    BASE "${CMAKE_CURRENT_SOURCE_DIR}/resources"
    FILES ${PLAYER_FRAME_FILES}
)
```

把猫咪动作帧图片加入 Qt 资源系统。  
加入后可以用这种路径读取：

```cpp
":/images/player/run_01.png"
```

```cmake
target_include_directories(running-maodie PRIVATE src)
```

把 `src` 目录加入头文件搜索路径。  
所以代码里可以写：

```cpp
#include "config/gameconfig.h"
```

而不用写完整相对路径。

```cmake
target_link_libraries(running-maodie PRIVATE Qt::Core Qt::Widgets)
```

把 Qt Core 和 Qt Widgets 链接到程序中。

## 四、`src/config/gameconfig.h`：游戏全局配置

位置：`src/config/gameconfig.h`

```cpp
namespace GameConfig {
```

定义命名空间，避免变量名污染全局空间。

```cpp
inline constexpr int WindowWidth = 1100;
inline constexpr int WindowHeight = 700;
```

游戏窗口基础尺寸。  
当前背景图高度也是 700，因此窗口高度和背景高度一致。

```cpp
inline constexpr int FrameIntervalMs = 16;
```

每 16 毫秒更新一帧，约等于 60 FPS。

```cpp
inline constexpr double Gravity = 0.8;
inline constexpr double JumpVelocity = -15.0;
```

- `Gravity` 是重力，每帧让竖直速度变大。
- `JumpVelocity` 是跳跃初速度。Qt 坐标系中 Y 轴向下，所以负数表示向上跳。

```cpp
inline constexpr int MaxJumpCount = 2;
```

允许二段跳。

```cpp
inline constexpr double HighFallThreshold = 160.0;
```

当腾空高度超过 160，并且角色正在下降时，动画从普通跳跃切换到 `fall`。

```cpp
inline constexpr double InitialWorldSpeed = 6.0;
```

世界向左移动的速度。  
背景、金币、障碍物都按这个速度向左滚动。

```cpp
inline constexpr int GroundLineY = 670;
```

地面线的 Y 坐标。  
如果你想调整地面高度，主要改这里。

```cpp
inline constexpr int PlayerX = 182;
```

玩家在屏幕上的固定 X 坐标。  
跑酷游戏里，猫咪不向右跑出屏幕，而是世界向左移动。

```cpp
inline constexpr int PlayerWidth = 48;
inline constexpr int PlayerHeight = 72;
```

猫咪物理碰撞箱尺寸，不等于图片尺寸。

```cpp
inline constexpr double PlayerVisualScale = 0.55;
```

猫咪图片绘制缩放比例。

```cpp
inline constexpr int GroundY = GroundLineY - PlayerHeight;
```

猫咪碰撞箱左上角的地面 Y 坐标。  
因为 `GroundLineY` 表示脚底位置，所以左上角要减去角色高度。

```cpp
inline constexpr int CoinSize = 28;
```

金币碰撞箱大小。

```cpp
inline constexpr int ObstacleMinGap = 300;
inline constexpr int ObstacleMaxGap = 560;
```

障碍物之间的随机间距范围。

```cpp
inline constexpr int ObstacleMinWidth = 210;
inline constexpr int ObstacleMaxWidth = 400;
inline constexpr int ObstacleHeight = 80;
```

火车障碍物的随机宽度范围和固定高度。

```cpp
inline constexpr int PlatformCoinSpacing = 100;
```

障碍物上方金币之间的水平间距。

```cpp
inline constexpr int ObstacleSideInset = 6;
inline constexpr int ObstacleTopInset = 4;
inline constexpr int ObstacleBottomInset = 3;
```

这样视觉矩形和实际碰撞判断不会过于苛刻。

```cpp
    pageStack->setCurrentWidget(gamePage);
```

```cpp
connect(gamePage, &GamePage::backToHomeRequested, this, [this] {
    homePage->resetHome();
    pageStack->setCurrentWidget(homePage);
});
```

当游戏页请求返回首页：

1. 重置首页状态。
2. 页面栈切回首页。

## 六、`HomePage`：首页、猫咪待机、入场动画

位置：

- `src/pages/homepage.h`
- `src/pages/homepage.cpp`

### 核心职责

`HomePage` 负责：

- 绘制首页动态背景。
- 显示顶部 Logo 预留区域。
- 显示“开始游戏”和“设置”按钮。
- 显示可点击猫咪。
- 长时间不点击猫咪时从 `sit` 进入 `sleep`。
- 点击开始后隐藏首页元素，让猫咪 `fall` 到地面，再跑到游戏起始位置。

### `homepage.h`

```cpp

```

首页是一个自绘控件，因此继承 `QWidget`。

```cpp

```


定义信号。  
当首页过渡动画完成后，发出这个信号通知主窗口进入游戏。

重写 Qt 事件函数：

- `resizeEvent`：窗口大小变化时重新摆放按钮。
- `mousePressEvent`：检测是否点击猫咪。


槽函数：

- `updateFrame`：定时器每帧调用。
- `beginGameTransition`：点击开始游戏后调用。

```cpp

    Sit,
    Sleep,
    Falling,
    MovingToGame,
    Finished
};
```

首页猫咪状态机。

- `Sit`：坐着。
- `Sleep`：睡觉。
- `Falling`：开始游戏后下落。
- `MovingToGame`：落地后跑到游戏位置。
- `Finished`：过渡完成，等待页面切换。

保存不同动作的图片帧。  
`QVector` 是 Qt 的动态数组，`QPixmap` 是 Qt 中用于显示图片的类。

```cpp
QHash<qint64, QRect> alphaBoundsCache;
```

缓存每张图片中非透明像素的边界。  
这样可以让不同尺寸 PNG 使用统一脚底锚点，避免动画切换时抖动。


首页 UI 和计时器对象。


首页运行状态：

- `state`：当前首页状态。
- `frameIndex`：当前动画帧序号。
- `frameElapsedMs`：当前帧已经播放多久。
- `inactiveElapsedMs`：猫咪多久没被点击。
- `backgroundOffset`：背景横向偏移。
- `catAnchor`：猫咪脚底中心锚点。
- `fallVelocity`：首页过渡时的下落速度。
- `catClickArea`：猫咪可点击区域。



```cpp
    return;
```

如果已经达到最大跳跃次数，就不再跳。

```cpp

    takeoffY = currentPosition.y();
```

如果是从地面起跳，记录起跳高度。

```cpp

    return;
```

如果在地面上，不受重力影响。

## 十一、`PlayerAnimator`：玩家动画状态机

位置：

- `src/animation/playeranimator.h`
- `src/animation/playeranimator.cpp`

### 核心职责

`PlayerAnimator` 只负责动画状态，不负责物理和绘制。

它根据这些信息决定播放什么动画：

- 是否在地面。
- 竖直速度。
- 水平速度。
- 腾空高度。
- 是否收到攻击/受伤请求。

### 状态枚举

```cpp
enum class PlayerAction {
    Idle,
    Walk,
    Run,
    Jump,
    Sit,
    Sleep,
    Attack,
    Hurt,
    Fall
};
```

定义所有动作状态。  
`enum class` 是强类型枚举，比普通 `enum` 更安全。

### 动画片段结构

```cpp
struct Clip {
    QVector<QPixmap> frames;
    int frameDurationMs = 100;
    bool loops = true;
};
```

每个动画片段包含：

- `frames`：图片帧数组。
- `frameDurationMs`：每帧持续时间。
- `loops`：是否循环。

### 构造函数

```cpp
addClip(PlayerAction::Idle,   "idle",   4, 140, true);
addClip(PlayerAction::Walk,   "walk",   7, 100, true);
addClip(PlayerAction::Run,    "run",    7, 75,  true);
...
```

注册每个动作对应的图片名、帧数、播放速度和是否循环。

例如：

```cpp
addClip(PlayerAction::Run, "run", 7, 75, true);
```

表示：

- 动作是 `Run`。
- 图片名前缀是 `run`。
- 有 7 帧。
- 每帧 75ms。
- 循环播放。

### 更新逻辑

```cpp
void PlayerAnimator::update(int deltaMs, bool grounded,
                            double verticalSpeed,
                            double horizontalSpeed,
                            double airborneHeight)
```

每帧更新动画。  
参数来自物理系统和游戏速度：

- `deltaMs`：这一帧经过的时间。
- `grounded`：是否在地面或平台。
- `verticalSpeed`：竖直速度。
- `horizontalSpeed`：水平速度。
- `airborneHeight`：腾空高度。

```cpp
if (currentAction == PlayerAction::Fall) { ... }
```

`fall` 优先级较高，先处理。

```cpp
if (terminalFall) {
    advanceTimedClip(deltaMs);
    return;
}
```

如果是死亡 fall，就只播放 fall，不再切回其他动作。

```cpp
if (grounded) {
    chooseLocomotion(true, horizontalSpeed);
    return;
}
```

如果非死亡 fall 已落地，则切回跑步/走路/待机。

```cpp
if (currentAction == PlayerAction::Hurt) { ... }
```

受伤动画优先于普通移动。

```cpp
if (currentAction == PlayerAction::Attack) { ... }
```

攻击动画播放期间不被普通移动打断。

```cpp
else if (previousFrame != 1 && frameIndex == 1) {
    attackActivationPending = true;
}
```

攻击动画切到第 2 帧时，标记攻击判定生效。  
数组下标从 0 开始，所以 `frameIndex == 1` 对应 `attack_02`。

```cpp
if (!grounded
    && verticalSpeed > 0.0
    && airborneHeight > GameConfig::HighFallThreshold) {
    transitionTo(PlayerAction::Fall);
    terminalFall = false;
    return;
}
```

如果角色在空中、正在下降、且腾空高度超过阈值，则切换到下落动画。

```cpp
if (!grounded) {
    if (currentAction != PlayerAction::Jump)
        transitionTo(PlayerAction::Jump);
    updateJumpFrame(verticalSpeed);
    inactiveElapsedMs = 0;
    return;
}
```

如果还在空中但不满足高空 fall，则播放 jump，并根据竖直速度选择具体帧。

```cpp
if (horizontalSpeed >= 4.0) {
    transitionTo(PlayerAction::Run);
}
```

速度较快时播放跑步。

```cpp
if (horizontalSpeed > 0.1) {
    transitionTo(PlayerAction::Walk);
}
```

速度较慢但不为 0 时播放走路。

```cpp
inactiveElapsedMs += deltaMs;
```

没有移动时累计无操作时间。

```cpp
if (inactiveElapsedMs >= 9000) {
    transitionTo(PlayerAction::Sleep);
} else if (inactiveElapsedMs >= 4000) {
    transitionTo(PlayerAction::Sit);
} else {
    transitionTo(PlayerAction::Idle);
}
```

无操作状态切换：

```text
idle -> sit -> sleep
```

### 攻击请求

```cpp
bool PlayerAnimator::requestAttack()
```

请求进入攻击状态。

```cpp
if (currentAction == PlayerAction::Fall
    || currentAction == PlayerAction::Hurt
    || currentAction == PlayerAction::Attack) {
    return false;
}
```

如果正在 fall、hurt、attack，则不允许新攻击打断。

```cpp
transitionTo(PlayerAction::Attack);
return true;
```

切换到攻击状态。

### 受伤和死亡

```cpp
void PlayerAnimator::requestHurt(int remainingHealth, bool strongKnockback)
```

请求受伤。

```cpp
if (remainingHealth <= 0 || strongKnockback) {
    transitionTo(PlayerAction::Fall);
    terminalFall = true;
    return;
}
```

生命归零或强击退时进入终结 fall。

```cpp
transitionTo(PlayerAction::Hurt);
```

普通受伤进入 hurt。

### 帧推进

```cpp
bool PlayerAnimator::advanceTimedClip(int deltaMs)
```

按时间推进动画帧。

```cpp
frameElapsedMs += deltaMs;
```

累计当前帧播放时间。

```cpp
while (frameElapsedMs >= clip.frameDurationMs)
```

如果累计时间超过单帧时长，就推进到下一帧。

```cpp
if (clip.loops) {
    frameIndex = 0;
    return false;
}
```

循环动画播放到最后后回到第 0 帧。

```cpp
frameIndex = clip.frames.size() - 1;
currentActionFinished = true;
return true;
```

非循环动画播放到最后后停在末帧，并标记动画结束。

## 十八、当前代码中的工程注意点

### 1. 绘制和逻辑已经分离

- `GameController` 管逻辑。
- `GameCanvas` 管绘制。
- `Player` 管物理。
- `PlayerAnimator` 管动画。

这个结构是正确方向。后续不要把碰撞判断写进 `paintEvent`。

### 2. 有些系统类目前是预留结构

`SpawnSystem`、`Collectible`、`AudioManager` 还没有完全接入主逻辑。  
这是正常的项目演进状态，后续可以逐步把 `GameController` 中的生成逻辑拆到 `SpawnSystem`。

### 3. 碰撞箱和图片尺寸不要混在一起

当前项目已经区分：

- 物理碰撞箱：`PlayerWidth`、`PlayerHeight`。
- 图片显示尺寸：PNG 原尺寸 × `PlayerVisualScale`。
- 对齐规则：非透明像素脚底中心对齐到碰撞箱脚底中心。

后续换素材时，优先保持这个规则。

### 4. Qt 坐标系和数学坐标系不同

Qt 中：

```text
左上角是 (0, 0)
X 向右增加
Y 向下增加
```

所以：

- 跳跃向上速度是负数。
- 重力是正数。
- 下落时 Y 坐标变大。
